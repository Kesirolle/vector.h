Detailed Description

Thread-safe abstract base class.

Anything derived from Referent can be stored in a ReferentPtr for shared ownership. The ReferentPtr class manages the reference count.

Referent is derived from Allocatable, so memory can be managed for any instance by passing an Allocator to operator new.

Copying of instances of classes derived from Referent is not allowed because reference-counted objects should not need to be copied. A derived class can implement a function to create and return a copy in special circumstances.

Note
All derived classes should declare their destructor as protected or private so that they cannot be created on the stack (which could easily result in double deletions).
